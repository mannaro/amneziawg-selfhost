    03. Установка и настройка Web Panel¶

    После установки сервера AmneziaWG управлять клиентами можно через веб-интерфейс. Панель позволяет создавать новые VPN-профили, удалять пользователей, просматривать состояние сервера и выполнять большинство операций без ручного редактирования конфигурационных файлов.

    В этой главе будет выполнена установка официальной Web Panel проекта AmneziaWG.

    Важно

    На данном этапе веб-панель будет доступна на сервере. Ограничение доступа и настройка межсетевого экрана будут рассмотрены в следующей главе.

    🎯 Цель¶

    После выполнения этой главы будет:

    • установлена Web Panel AmneziaWG;
    • создана служба systemd;
    • настроен автоматический запуск панели;
    • проверена доступность веб-интерфейса;
    • подготовлена система к дальнейшей настройке Firewall.

    📋 Требования¶

    Перед началом убедитесь, что:

    • выполнена глава 02. Установка AmneziaWG;
    • VPN-интерфейс awg0 успешно создан;
    • служба awg-quick@awg0 работает;
    • сервер имеет доступ в Интернет;
    • имеется пользователь root или права sudo.

    Проверьте состояние VPN:

    systemctl status awg-quick@awg0 --no-pager

    Ожидаемый результат:

    Active: active (exited)

    Также рекомендуется убедиться, что интерфейс существует:

    ip -brief address show awg0

    📁 Переход в каталог установщика¶

    Если каталог сохранился после предыдущей главы:

    cd /root/setup/installers/amneziawg-install

    Если каталог был удалён:

    cd /root/setup/installers

git clone https://github.com/wiresock/amneziawg-install.git

cd amneziawg-install

    Проверьте содержимое:

    ls -la

    Должен присутствовать файл:

    amneziawg-web.sh

    📌 Проверка версии¶

    Перед установкой рекомендуется убедиться, что используется ожидаемая версия проекта.

    Получить текущий commit:

    git rev-parse HEAD

    Посмотреть доступные теги:

    git tag --sort=-version:refname | head

    Если документация рассчитана на конкретную версию:

    git checkout web-v0.1.8

    После переключения убедитесь:

    git status
git describe --tags --always

    Примечание

    Использование фиксированной версии позволяет избежать неожиданного изменения поведения панели после выхода новых релизов.

    🚀 Запуск установщика¶

    Сделайте файл исполняемым:

    chmod +x amneziawg-web.sh

    Запустите установку:

    ./amneziawg-web.sh

    или

    sudo ./amneziawg-web.sh

    Во время установки автоматически выполняются следующие действия:

    1. установка необходимых зависимостей;
    2. загрузка компонентов Web Panel;
    3. создание рабочего каталога;
    4. регистрация службы systemd;
    5. запуск веб-сервера;
    6. включение автозапуска после перезагрузки.

    Во время установки внимательно следите за сообщениями установщика. При успешном завершении будут выведены адрес панели и параметры для первого входа.

    📂 Проверка установленных файлов¶

    После завершения установки убедитесь, что необходимые файлы появились в системе.

    Проверьте расположение компонентов:

    find /opt -maxdepth 2 -iname "*amnezia*" -o -iname "*web*"

    Если установщик использует другой каталог, определите его командой:

    systemctl cat amneziawg-web

    Именно этот каталог далее будет использоваться при обновлении и резервном копировании панели.

    ⚙ Проверка службы¶

    После завершения установки необходимо убедиться, что служба Web Panel успешно зарегистрирована в systemd.

    Просмотрите её состояние:

    systemctl status amneziawg-web --no-pager

    При успешном запуске вывод должен содержать примерно следующее:

    Loaded: loaded
Active: active (running)

    Если служба находится в состоянии failed, сразу просмотрите журнал:

    journalctl -u amneziawg-web -n 50 --no-pager

    Большинство проблем на этом этапе связано с:

    • отсутствием необходимых зависимостей;
    • ошибками во время установки;
    • занятым сетевым портом;
    • повреждением файлов панели.

    🚀 Автоматический запуск¶

    Проверьте, включён ли автозапуск службы:

    systemctl is-enabled amneziawg-web

    Ожидаемый результат:

    enabled

    Если служба отключена:

    systemctl enable amneziawg-web

    🌐 Проверка прослушиваемого порта¶

    Определите, какой TCP-порт использует Web Panel.

    Самый простой способ:

    ss -ltnp

    или

    ss -tulpn | grep LISTEN

    В выводе должен появиться процесс панели с открытым TCP-портом.

    Например:

    LISTEN
0.0.0.0:8000

    Именно этот порт далее будет использоваться для подключения через браузер.

    Примечание

    В разных версиях установщика номер порта может отличаться.

    🌍 Первый вход¶

    Откройте браузер и перейдите по адресу:

    http://IP_ВАШЕГО_СЕРВЕРА:PORT

    Например:

    http://203.0.113.10:8000

    Если панель открылась успешно, появится окно авторизации.

    Во время первого запуска установщик выводит:

    • адрес панели;
    • логин;
    • пароль.

    Рекомендуется сохранить эти данные в безопасном месте.

    🔑 Смена пароля¶

    Если используется пароль, автоматически созданный установщиком, замените его сразу после первого входа.

    Используйте длинный уникальный пароль.

    Рекомендуется хранить его в менеджере паролей (например, Vaultwarden).

    🔎 Проверка доступности¶

    Проверьте, что панель отвечает на HTTP-запросы.

    С самого сервера:

    curl http://127.0.0.1:PORT

    Если панель работает, будет возвращён HTML-код страницы авторизации.

    Если используется внешний IP:

    curl http://SERVER_IP:PORT

    При отсутствии ответа проверьте:

    • работает ли служба;
    • открыт ли порт;
    • нет ли ошибок Firewall;
    • не занят ли порт другим процессом.

    📜 Просмотр журналов¶

    Полный журнал службы:

    journalctl -u amneziawg-web --no-pager

    Последние 100 строк:

    journalctl -u amneziawg-web -n 100 --no-pager

    Наблюдение за журналом в реальном времени:

    journalctl -u amneziawg-web -f

    Для выхода нажмите:

    Ctrl+C

    Журнал — первое место, куда стоит смотреть при возникновении любых ошибок панели.

    🔄 Перезапуск панели¶

    После изменения настроек можно перезапустить службу:

    systemctl restart amneziawg-web

    Проверить её состояние:

    systemctl status amneziawg-web --no-pager

    При необходимости полностью остановить службу:

    systemctl stop amneziawg-web

    Запустить снова:

    systemctl start amneziawg-web
    ¶

    ⚠️ Типичные ошибки¶

    Служба не запускается¶

    Если systemctl status показывает состояние failed, выполните:

    journalctl -u amneziawg-web -n 100 --no-pager

    Чаще всего причиной являются:

    • ошибка во время установки;
    • отсутствующие зависимости;
    • неверные права доступа;
    • повреждённые файлы панели.

    Порт уже используется¶

    Если панель не запускается, проверьте занятые порты:

    ss -tulpn

    или

    lsof -i -P -n

    Если нужный порт уже используется другим процессом, измените порт панели или остановите конфликтующую службу.

    Панель открывается только локально¶

    Если curl http://127.0.0.1:PORT работает, а из браузера панель недоступна:

    проверьте:

    • Firewall VPS;
    • Security Group (если используется облачный провайдер);
    • правила nftables;
    • открыт ли TCP-порт.

    В следующей главе будет настроено безопасное открытие доступа к панели.

    После обновления панель перестала работать¶

    Проверьте журнал:

    journalctl -u amneziawg-web

    Убедитесь, что:

    • используется совместимая версия Web Panel;
    • служба успешно перезапущена;
    • все файлы установлены корректно.

    При необходимости повторно выполните установочный скрипт.

    🔄 Обновление Web Panel¶

    Перед обновлением рекомендуется создать резервную копию конфигурации.

    Перейдите в каталог проекта:

    cd /root/setup/installers/amneziawg-install

    Получите последние изменения:

    git pull

    При использовании фиксированной версии:

    git fetch --tags
git checkout web-v0.1.8

    Запустите установщик повторно:

    ./amneziawg-web.sh

    После завершения обновления перезапустите службу:

    systemctl restart amneziawg-web

    Проверьте её состояние:

    systemctl status amneziawg-web --no-pager

    🗑 Удаление панели¶

    Если потребуется удалить Web Panel, сначала остановите службу:

    systemctl stop amneziawg-web

    Отключите автозапуск:

    systemctl disable amneziawg-web

    Дальнейшее удаление зависит от используемой версии установщика.

    Перед удалением рекомендуется сохранить резервную копию всех конфигурационных файлов.

    ✅ Итоговая проверка¶

    Перед переходом к следующей главе выполните:

    systemctl is-enabled amneziawg-web
systemctl is-active amneziawg-web
ss -ltnp

    Убедитесь, что:

    • ✅ Web Panel успешно установлена;
    • ✅ служба зарегистрирована в systemd;
    • ✅ автозапуск включён;
    • ✅ служба находится в состоянии active (running);
    • ✅ веб-интерфейс открывается в браузере;
    • ✅ известен используемый TCP-порт;
    • ✅ журналы доступны через journalctl.

    🎉 Итог¶

    На этом этапе Web Panel полностью установлена и готова к использованию.

    Панель позволяет управлять клиентами AmneziaWG через веб-интерфейс без ручного редактирования конфигурационных файлов.

    Пока что панель доступна по открытому TCP-порту. Для повышения безопасности рекомендуется ограничить доступ к ней и настроить фильтрацию трафика.

    ➡️ Следующая глава¶

    На следующем этапе мы настроим межсетевой экран с использованием nftables, чтобы защитить сервер и ограничить доступ только к необходимым сервисам.

    Будут рассмотрены:

    • создание базового набора правил;
    • защита SSH;
    • открытие только необходимых TCP и UDP портов;
    • ограничение доступа к Web Panel;
    • сохранение правил после перезагрузки системы.
