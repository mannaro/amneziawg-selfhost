# 05. Split Routing

## Что такое Split Routing

Split Routing (или Split Tunnel) — это режим работы VPN, при котором через VPN-туннель направляется только определённый трафик, а остальной продолжает использовать обычное интернет-соединение.

В отличие от Full Tunnel, где весь сетевой трафик проходит через VPN, Split Routing позволяет выбирать, какие ресурсы должны использовать защищённый канал.

Это уменьшает задержки, снижает нагрузку на VPS и позволяет одновременно пользоваться локальными ресурсами домашней сети и VPN.

---

## Full Tunnel и Split Tunnel

### Full Tunnel

```
                Интернет
                    │
                    ▼
                 VPS/VPN
                    ▲
                    │
          Весь сетевой трафик
                    ▲
                    │
                 Устройство
```

Преимущества:

- весь трафик проходит через VPN;
- единый внешний IP;
- максимальная приватность.

Недостатки:

- увеличивается задержка;
- возрастает нагрузка на VPS;
- локальная сеть становится менее доступной;
- весь интернет зависит от работоспособности VPN.

---

### Split Tunnel

```
                    Интернет
                 ▲            ▲
                 │            │
          VPN-трафик      Обычный трафик
                 │            │
                 ▼            ▼
              VPS         Провайдер
                 ▲            ▲
                  \          /
                   \        /
                    Устройство
```

Преимущества:

- VPN используется только там, где это необходимо;
- меньше задержка;
- меньше расходуется трафик VPS;
- локальная сеть продолжает работать напрямую.

Именно такой вариант используется в данном проекте.

---

# Где принимается решение

Важно понимать один момент.

Сервер AmneziaWG **не принимает решение**, какой трафик должен идти через VPN.

Сервер просто получает пакеты от клиента и пересылает их в Интернет.

Вся логика Split Routing находится на стороне клиентов.

В данном проекте используются два разных подхода:

- Windows-клиенты используют маршруты (`AllowedIPs`);
- роутер Cudy использует Podkop, policy routing и sing-box.

# Split Routing на Windows

На Windows используется классический механизм маршрутизации.

После подключения клиента появляются дополнительные маршруты через VPN-интерфейс.

При этом основной шлюз системы остаётся домашним маршрутизатором.

Получается следующая схема:

```
Локальная сеть
        │
        ▼
Домашний роутер
        │
        ├──────────────► Обычный Интернет
        │
        └──────────────► VPN-маршруты
                          │
                          ▼
                     AmneziaWG
                          │
                          ▼
                         VPS
```

Это означает, что:

- обычные сайты открываются напрямую;
- только нужные сети используют VPN.

---

# Split Routing на Cudy OpenWrt

Для маршрутизатора используется более гибкая схема.

Вместо обычных маршрутов применяется несколько компонентов:

- Podkop;
- nftables;
- Policy Routing;
- sing-box;
- AmneziaWG.

Такая схема позволяет маршрутизировать трафик не только по IP-адресам, но и по доменным именам.

---

# Как работает маршрутизация

Последовательность обработки выглядит следующим образом.

```
Устройство
      │
      ▼
Cudy OpenWrt
      │
      ▼
Podkop определяет,
нужно ли использовать VPN
      │
      ▼
nftables устанавливает fwmark
      │
      ▼
Policy Routing
      │
      ▼
таблица podkop
      │
      ▼
TPROXY
      │
      ▼
sing-box
      │
      ▼
AmneziaWG
      │
      ▼
VPS
      │
      ▼
Интернет
```

Именно эта архитектура используется в данной системе.

---

# Почему используется sing-box

Многие VPN-решения работают исключительно через изменение маршрутов.

В данном проекте используется другой подход.

После определения нужного трафика Podkop передаёт его в локальный экземпляр sing-box с использованием механизма TPROXY.

После этого sing-box самостоятельно отправляет соединение через интерфейс AmneziaWG.

Такой подход значительно гибче обычной маршрутизации.

Он позволяет:

- работать с доменными именами;
- автоматически обновлять списки;
- использовать большие наборы адресов;
- не изменять основной маршрут системы.

---

# Роль VPS

Сервер не знает, какой именно трафик должен использовать VPN.

Он выполняет только следующие задачи:

- принимает соединения AmneziaWG;
- расшифровывает пакеты;
- пересылает их в Интернет;
- выполняет NAT (masquerade);
- отправляет ответы обратно клиенту.

Таким образом, вся логика выбора трафика находится исключительно на стороне клиентов.

---

# Преимущества такой архитектуры

Используемая схема имеет несколько преимуществ.

- основной интернет продолжает работать напрямую;
- VPN используется только там, где это необходимо;
- отсутствует лишняя нагрузка на VPS;
- домашняя сеть остаётся доступной;
- удобно обновлять списки маршрутизации;
- можно легко масштабировать систему.
