# 01. Подготовка VPS

В этой главе подготавливается чистый сервер Ubuntu к установке AmneziaWG. После выполнения всех шагов сервер будет соответствовать требованиям для дальнейшей настройки VPN-сервера.

---

# 🎯 Цель

Подготовить VPS к установке AmneziaWG:

- проверить операционную систему;
- обновить сервер;
- установить необходимые утилиты;
- настроить синхронизацию времени;
- включить маршрутизацию IPv4;
- проверить сетевую конфигурацию;
- убедиться, что выбранный UDP-порт свободен.

---

# 📋 Требования

Минимальная рекомендуемая конфигурация VPS:

- Ubuntu 22.04 LTS или Ubuntu 24.04 LTS;
- 1 виртуальное ядро (vCPU);
- 512 МБ–1 ГБ оперативной памяти;
- 5–10 ГБ SSD;
- публичный IPv4-адрес;
- доступ по SSH с правами `root` или пользователя с `sudo`.

> **Рекомендация**
>
> По возможности выбирайте VPS в стране, из которой доступны необходимые сайты и сервисы. Это может упростить дальнейшую работу с сервером.

---

# 🔒 Безопасность

Никогда не публикуйте в репозитории:

- приватные ключи;
- клиентские конфигурации;
- пароли;
- резервные архивы;
- реальные IP-адреса, если они не являются частью примера.

Во всех примерах используйте обезличенные значения, например:

```text
YOUR_SERVER_IP
```

---

# 🚀 Подключение по SSH

Подключитесь к серверу:

```bash
ssh root@YOUR_SERVER_IP
```

При первом подключении SSH попросит подтвердить отпечаток сервера:

```text
The authenticity of host can't be established.
Are you sure you want to continue connecting (yes/no)?
```

После проверки IP-адреса подтвердите подключение:

```text
yes
```

После этого отпечаток будет сохранён в:

```text
~/.ssh/known_hosts
```

---

# 🖥 Проверка установленной системы

Убедитесь, что сервер работает под управлением поддерживаемой версии Ubuntu.

```bash
cat /etc/os-release
```

Ожидаемый результат:

```text
NAME="Ubuntu"
VERSION="24.04 LTS"
```

Проверьте архитектуру процессора:

```bash
uname -m
```

Для большинства VPS ожидается:

```text
x86_64
```

---

# ⬆ Обновление системы

Перед установкой нового программного обеспечения рекомендуется обновить систему.

```bash
apt update
apt upgrade -y
```

Если было обновлено ядро Linux или важные системные компоненты, выполните перезагрузку:

```bash
reboot
```

После перезагрузки снова подключитесь по SSH.

---

# 📦 Установка необходимых пакетов

Установите базовый набор утилит:

```bash
apt install -y \
  curl \
  wget \
  git \
  jq \
  ca-certificates \
  gnupg \
  unzip \
  tar \
  nftables
```

Используемые пакеты:

| Пакет | Назначение |
|-------|------------|
| `curl`, `wget` | загрузка файлов и проверка веб-сервисов |
| `git` | получение исходного кода |
| `jq` | обработка JSON |
| `ca-certificates` | проверка TLS-сертификатов |
| `tar`, `unzip` | работа с архивами |
| `nftables` | настройка firewall |

---

# 🕒 Проверка системного времени

Корректное системное время необходимо для работы TLS-сертификатов и сетевых сервисов.

Проверьте статус:

```bash
timedatectl
```

Убедитесь, что присутствуют строки:

```text
System clock synchronized: yes
NTP service: active
```

При необходимости включите синхронизацию:

```bash
timedatectl set-ntp true
```

---

# 🌐 Проверка сети

Просмотрите сетевые интерфейсы:

```bash
ip -brief address
```

Проверьте маршрут по умолчанию:

```bash
ip route
```

Определите внешний сетевой интерфейс:

```bash
ip route show default | awk '{print $5; exit}'
```

Обычно интерфейс называется:

- `eth0`
- `ens3`
- `ens18`

или аналогичным образом.

> Название интерфейса понадобится позже при настройке NAT.

---

# 🌍 Проверка публичного IPv4

Убедитесь, что сервер использует ожидаемый публичный IP-адрес.

```bash
curl -4 https://ifconfig.me
echo
```

Полученный адрес должен совпадать с указанным в панели управления VPS.

Во всех примерах документации вместо реального IP используйте:

```text
YOUR_SERVER_IP
```

---

# 🔀 Включение IPv4 Forwarding

Для работы VPN сервер должен пересылать трафик между VPN-интерфейсом и интернетом.

Создайте файл конфигурации:

```bash
cat > /etc/sysctl.d/99-amneziawg.conf <<'EOF'
net.ipv4.ip_forward = 1
EOF
```

Примените настройки:

```bash
sysctl --system
```

Проверьте результат:

```bash
sysctl net.ipv4.ip_forward
```

Ожидаемый вывод:

```text
net.ipv4.ip_forward = 1
```

---

# 🔌 Проверка свободных портов

Посмотрите все используемые TCP и UDP порты:

```bash
ss -tulpn
```

Проверьте конкретный UDP-порт:

```bash
ss -lunp | grep ':51820'
```

Если команда ничего не вывела, порт свободен.

В этой документации используем обозначение:

```text
AWG_PORT
```

Это означает выбранный свободный UDP-порт для AmneziaWG.

---

Перед переходом к следующему разделу выполните:

```bash

```

- ✅ система обновлена;

- ✅ VPS имеет публичный IPv4;
- ✅ включён IPv4 Forwarding;
- ✅ определён внешний сетевой интерфейс;
- ✅ выбран свободный UDP-порт.
