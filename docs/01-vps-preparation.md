# 1. Подготовка VPS

В этом разделе подготавливается чистый сервер Ubuntu для установки AmneziaWG.

## Требования

Минимальная конфигурация VPS:

- Ubuntu 22.04 LTS или Ubuntu 24.04 LTS;
- 1 виртуальное ядро;
- 512 МБ–1 ГБ оперативной памяти;
- 5–10 ГБ дискового пространства;
- публичный IPv4-адрес;
- доступ по SSH с правами `root` или через `sudo`.

Сервер желательно размещать в стране, из которой доступны необходимые сайты и сервисы.

> [!IMPORTANT]
> Не публикуйте в GitHub реальные приватные ключи, пароли, клиентские конфигурации и резервные архивы.

## Подключение по SSH

Подключитесь к серверу:

```bash
ssh root@YOUR_SERVER_IP
```

При первом подключении SSH предложит проверить ключ сервера:

```text
The authenticity of host cannot be established.
Are you sure you want to continue connecting?
```

После проверки IP-адреса подтвердите подключение:

```text
yes
```

Отпечаток сервера будет сохранён локально в `~/.ssh/known_hosts`.

## Проверка Ubuntu

Проверьте установленную систему:

```bash
cat /etc/os-release
```

Пример:

```text
NAME="Ubuntu"
VERSION="24.04 LTS"
```

Проверьте архитектуру:

```bash
uname -m
```

Для большинства VPS ожидается:

```text
x86_64
```

## Обновление системы

```bash
apt update
apt upgrade -y
```

Если обновилось ядро или системные компоненты, перезагрузите VPS:

```bash
reboot
```

После перезагрузки снова подключитесь по SSH.

## Установка базовых пакетов

```bash
apt install -y \
  curl \
  wget \
  git \
  jq \
  ca-certificates \
  gnupg \
  unzip \
  tar \
  nftables
```

Назначение основных пакетов:

- `curl` и `wget` — загрузка файлов и проверка веб-сервисов;
- `git` — получение исходного кода;
- `jq` — обработка JSON;
- `ca-certificates` — проверка TLS-сертификатов;
- `tar` и `unzip` — работа с архивами;
- `nftables` — firewall и фильтрация трафика.

## Проверка системного времени

```bash
timedatectl
```

Проверьте наличие строк:

```text
System clock synchronized: yes
NTP service: active
```

При необходимости включите синхронизацию:

```bash
timedatectl set-ntp true
```

## Проверка сети

Покажите сетевые интерфейсы:

```bash
ip -brief address
```

Проверьте маршрут по умолчанию:

```bash
ip route
```

Автоматически определить внешний интерфейс можно так:

```bash
ip route show default | awk '{print $5; exit}'
```

Обычно он называется `eth0`, `ens3`, `ens18` или похожим образом.

Название внешнего интерфейса потребуется при настройке NAT.

## Проверка публичного IPv4

```bash
curl -4 https://ifconfig.me
echo
```

Адрес должен совпадать с IPv4, указанным в панели VPS-провайдера.

В публичной документации вместо реального адреса используйте:

```text
YOUR_SERVER_IP
```

## Включение IPv4 forwarding

VPN-сервер должен пересылать пакеты между VPN-интерфейсом и интернетом.

Создайте постоянную настройку:

```bash
cat > /etc/sysctl.d/99-amneziawg.conf <<'EOF'
net.ipv4.ip_forward = 1
EOF
```

Примените параметры:

```bash
sysctl --system
```

Проверьте результат:

```bash
sysctl net.ipv4.ip_forward
```

Ожидаемый вывод:

```text
net.ipv4.ip_forward = 1
```

## Проверка занятых портов

```bash
ss -tulpn
```

Проверка конкретного UDP-порта:

```bash
ss -lunp | grep ':51820'
```

Если команда ничего не вывела, этот порт сейчас не занят.

AmneziaWG может использовать другой свободный UDP-порт. В примерах документации он обозначается как:

```text
AWG_PORT
```

## Итоговая проверка

```bash
cat /etc/os-release
uname -m
ip -brief address
ip route
sysctl net.ipv4.ip_forward
timedatectl
ss -tulpn
```

Перед продолжением убедитесь, что:

- установлена поддерживаемая версия Ubuntu;
- системные пакеты обновлены;
- системное время синхронизировано;
- VPS имеет публичный IPv4;
- включена маршрутизация IPv4;
- определён внешний сетевой интерфейс;
- выбран свободный UDP-порт.

Далее: [Установка AmneziaWG](02-amneziawg-installation.md).
