# 09. Podkop и выборочная маршрутизация

## Введение

В предыдущей главе маршрутизатор Cudy был успешно подключён к серверу AmneziaWG.

Однако одного VPN-подключения недостаточно.

Если направить через VPN абсолютно весь трафик, увеличится задержка, возрастёт нагрузка на VPS, а доступ к локальным ресурсам станет менее удобным.

Именно поэтому в данном проекте используется система **Podkop**, позволяющая направлять через VPN только необходимый трафик.

В этой главе подробно рассматривается архитектура этой системы.

---

# Что такое Podkop

Podkop — это система выборочной маршрутизации для OpenWrt.

Она анализирует соединения и принимает решение:

- отправить трафик напрямую;
- либо передать его в VPN.

Главная идея состоит в том, что VPN используется только там, где он действительно необходим.

Благодаря этому достигается высокая скорость работы сети при сохранении возможности обхода ограничений.

---

# Общая архитектура

После полной настройки движение пакетов выглядит следующим образом.

```
                Устройство

                     ▼
             Cudy OpenWrt
                     │
                     ▼
               Podkop
                     │
          принимает решение
                     │

```

Именно такая схема используется в данном проекте.

---

# Почему используется именно такая схема

Можно было бы просто изменить маршрут по умолчанию и отправить весь трафик через VPN.

Но у такого подхода есть недостатки:

- весь Интернет зависит от VPN;
- увеличивается задержка;
- возрастает нагрузка на сервер;
- сложнее использовать локальную сеть.

Использование Podkop позволяет избежать этих ограничений.

---

# Как принимается решение

Podkop работает ещё до того, как пакет попадёт в VPN.

Он определяет, требуется ли использовать VPN для конкретного соединения.

Если VPN не требуется, пакет сразу отправляется через обычного интернет-провайдера.

Если VPN необходим, пакет передаётся следующему этапу обработки.

Важно понимать, что Podkop **не передаёт пакет непосредственно в интерфейс AmneziaWG**.

Между принятием решения и VPN существует ещё несколько компонентов.

---

# Роль nftables

После того как Podkop определил необходимость использования VPN, пакет попадает в правила **nftables**.

Здесь пакету присваивается специальная метка (**fwmark**).

В данной конфигурации используется значение:

```text
0x100000
```

Само по себе присвоение метки ничего не маршрутизирует.

Она лишь сообщает ядру Linux, что данный пакет необходимо обработать по специальным правилам.

---

# Policy Routing

После установки fwmark начинает работать механизм Policy Routing.

Вместо основной таблицы маршрутизации используется отдельная таблица.

В данной конфигурации она называется:

```text
podkop
```

Проверить это можно командой:

```bash
ip rule
```

Среди правил можно увидеть:

```text
105: from all fwmark 0x100000 lookup podkop
```

Это означает:

> Все пакеты с меткой `0x100000` должны использовать таблицу маршрутизации `podkop`.

Именно здесь начинается принципиальное отличие от обычной маршрутизации Linux.

---

# Таблица маршрутизации podkop

Если посмотреть содержимое таблицы маршрутизации, можно увидеть необычный маршрут.

Проверить его можно командой:

```bash
ip route show table podkop
```

В данной конфигурации вывод выглядит следующим образом:

```text
local default dev lo scope host
```

На первый взгляд кажется, что это ошибка.

На самом деле именно так и должна работать схема с TPROXY.

Вместо отправки пакета в интерфейс VPN ядро Linux направляет его в локальный интерфейс (`lo`).

Дальнейшую обработку выполняет уже пользовательское приложение.

---

# Что такое TPROXY

TPROXY (Transparent Proxy) — механизм ядра Linux, позволяющий перехватывать сетевые соединения без изменения приложения, которое их создаёт.

С точки зрения программы соединение устанавливается как обычно.

Однако ядро незаметно перенаправляет пакет локальному прокси.

В данной конфигурации именно TPROXY передаёт пакеты процессу `sing-box`.

---

# Роль sing-box

После попадания в TPROXY пакет получает приложение `sing-box`.

Именно оно принимает окончательное решение, каким исходящим интерфейсом воспользоваться.

В нашем проекте используется интерфейс:

```text
awg0
```

Таким образом пакет проходит путь:

```
Приложение
        │
        ▼
TPROXY
        │
        ▼
sing-box
        │
        ▼
AmneziaWG
```

Именно поэтому в таблице маршрутизации отсутствует обычный маршрут через VPN-интерфейс.

---

# Где работает sing-box

Проверить его можно командой:

```bash
netstat -lntup
```

В нашей конфигурации можно увидеть:

```text
127.0.0.1:1602
```

Именно на этом локальном адресе sing-box ожидает соединения от TPROXY.

Эта служба недоступна из локальной сети и используется исключительно внутри самого маршрутизатора.

---

# Диагностика

Большинство компонентов можно проверить стандартными средствами OpenWrt.

---

## Проверка интерфейса AmneziaWG

```bash
ip -br addr show awg0
```

Ожидаемый результат:

```text
awg0    UP    10.66.66.4/32
```

---

## Проверка Handshake

```bash
awg show
```

Необходимо убедиться, что строка:

```text
latest handshake
```

регулярно обновляется.

---

## Проверка Policy Routing

```bash
ip rule
```

В выводе должно присутствовать правило:

```text
fwmark 0x100000 lookup podkop
```

Это означает, что Policy Routing работает корректно.

---

## Проверка таблицы podkop

```bash
ip route show table podkop
```

Ожидаемый вывод:

```text
local default dev lo scope host
```

Именно этот маршрут используется механизмом TPROXY.

---

## Проверка sing-box

```bash
ps | grep sing-box
```

Если процесс присутствует в списке, значит служба запущена.

Дополнительно можно убедиться, что она прослушивает локальный порт:

```bash
netstat -lntup | grep 1602
```

---

# Почему это лучше обычного VPN

Используемая архитектура обладает рядом преимуществ.

- Не изменяется основной маршрут системы.
- Домашняя сеть продолжает работать напрямую.
- Через VPN проходит только необходимый трафик.
- Легко обновляются списки маршрутизации.
- Возможно использование правил по доменным именам.
- VPN не влияет на работу остальных устройств сети.

Именно поэтому данная схема считается одной из наиболее гибких для домашнего использования.
