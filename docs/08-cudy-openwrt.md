# 08. Настройка Cudy OpenWrt

## Введение

В предыдущих главах были рассмотрены клиенты для Windows и iPhone.

Однако наиболее удобным вариантом является подключение VPN непосредственно на маршрутизаторе.

В этом случае все устройства домашней сети могут использовать VPN без установки дополнительных приложений.

В данном проекте используется маршрутизатор **Cudy WR3000S** с прошивкой **OpenWrt**.

После выполнения данной главы роутер будет подключаться к серверу AmneziaWG автоматически.

---

# Почему OpenWrt

OpenWrt предоставляет значительно больше возможностей по сравнению со штатной прошивкой большинства домашних маршрутизаторов.

Основные преимущества:

- полный доступ к Linux;
- пакетный менеджер `opkg`;
- поддержка nftables;
- гибкая маршрутизация;
- возможность запуска дополнительных сервисов;
- поддержка AmneziaWG.

Именно поэтому OpenWrt хорошо подходит для построения собственного VPN-шлюза.

---

# Схема подключения

После настройки архитектура сети будет выглядеть следующим образом.

```
                    Интернет
                         │
                         ▼
                     VPS AmneziaWG
                         ▲
                         │
                  Зашифрованный туннель
                         ▲
                         │
                 Cudy OpenWrt Router
                         ▲
                         │
        ┌────────────┬────────────┬────────────┐
        │            │            │
     Компьютер     Телефон        ТВ
```

Все устройства продолжают использовать домашний роутер как основной шлюз.

При этом сам роутер принимает решение, какой трафик необходимо отправить через VPN.

---

# Подготовка OpenWrt

Перед началом рекомендуется обновить список пакетов.

```bash
opkg update
```

Также желательно убедиться, что система имеет актуальную дату и время.

Проверить можно командой:

```bash
date
```

---

# Установка AmneziaWG

Установите необходимые пакеты.

```bash
opkg install amneziawg-tools luci-proto-amneziawg
```

После завершения установки появится поддержка интерфейса AmneziaWG как в командной строке, так и в веб-интерфейсе LuCI.

---

# Создание клиента

Откройте веб-панель AmneziaWG на сервере.

Создайте нового клиента.

Например:

```
Cudy Router
```

После создания экспортируйте конфигурацию.

Для роутера удобнее использовать обычный конфигурационный файл.

---

# Импорт конфигурации

Откройте LuCI.

Перейдите:

```
Network
→ Interfaces
```

Создайте новый интерфейс AmneziaWG.

Используйте параметры из экспортированного конфигурационного файла.

После сохранения должен появиться новый интерфейс.

---

# Проверка интерфейса

После запуска подключитесь к роутеру по SSH и выполните:

```bash
ip -br addr show awg0
```

Ожидаемый результат:

```text
awg0    UP    10.66.66.4/32
```

Если интерфейс появился, конфигурация импортирована успешно.

---

# Проверка Handshake

Проверьте состояние интерфейса.

```bash
awg show
```

Ожидаемый результат:

```text
latest handshake: 6 seconds ago
transfer: ...
```

Постоянное обновление Handshake означает успешное подключение к серверу.

---

# Автоматический запуск

Чтобы VPN автоматически поднимался после перезагрузки роутера, необходимо включить автоматический запуск интерфейса.

В LuCI откройте настройки интерфейса AmneziaWG и убедитесь, что включена опция:

```
Bring up on boot
```

После этого соединение будет устанавливаться автоматически при загрузке OpenWrt.

---

# Проверка подключения

После настройки рекомендуется выполнить несколько простых проверок.

## Проверка интерфейса

```bash
ip -br addr show awg0
```

Ожидаемый результат:

```text
awg0    UP    10.66.66.4/32
```

---

## Проверка Handshake

```bash
awg show
```

Пример успешного подключения:

```text
interface: awg0
  listening port: 48618

peer: XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
  endpoint: your-vps.example.com:53925
  latest handshake: 14 seconds ago
  transfer: 28.4 MiB received, 91.3 MiB sent
```

Особенно важно, чтобы строка `latest handshake` регулярно обновлялась.

---

## Проверка связи с сервером

Попробуйте открыть веб-панель AmneziaWG по VPN-адресу сервера.

```
http://10.66.66.1:8080
```

Если панель открывается, значит:

- VPN работает;
- маршрутизация функционирует;
- соединение между роутером и VPS установлено.

---

# Проверка после перезагрузки

Перезагрузите маршрутизатор.

```bash
reboot
```

После загрузки снова выполните:

```bash
awg show
```

Если Handshake появился автоматически, значит интерфейс успешно запускается без участия пользователя.

---

# Типичные проблемы

## Интерфейс awg0 отсутствует

Проверьте:

```bash
ip link show
```

Если интерфейс отсутствует:

- убедитесь, что пакет AmneziaWG установлен;
- проверьте правильность конфигурации;
- перезапустите сетевую службу.

---

## Нет Handshake

Если вывод выглядит примерно так:

```text
latest handshake: never
```

возможные причины:

- VPS недоступен;
- неверный Endpoint;
- закрыт UDP-порт;
- ошибка в ключах;
- неправильное время на устройстве.

Первым делом убедитесь, что сервер отвечает и UDP-порт открыт.

---

## Нет доступа к панели управления

Если VPN подключён, но страница

```
http://10.66.66.1:8080
```

не открывается, проверьте:

- подключение VPN;
- правила nftables на сервере;
- VPN-адрес роутера;
- наличие Handshake.

---

## Интернет работает, но VPN не используется

Это нормальная ситуация.

На данном этапе роутер только подключён к серверу AmneziaWG.

Маршрутизация трафика через VPN ещё не настроена.

Этим мы займёмся в следующей главе.

---

# Что ещё предстоит настроить

После завершения данной главы роутер умеет:

- подключаться к серверу;
- автоматически восстанавливать соединение;
- передавать данные через туннель;
- работать как полноценный клиент AmneziaWG.

Однако пока весь трафик продолжает использовать обычное подключение к Интернету.

Чтобы направлять через VPN только необходимые сайты и сервисы, потребуется настроить систему выборочной маршрутизации.
