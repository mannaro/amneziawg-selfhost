# 10. Резервное копирование и восстановление

## Введение

Даже идеально настроенная система не защищена от сбоев.

Причиной восстановления могут стать:

- переустановка VPS;
- повреждение файловой системы;
- замена маршрутизатора;
- ошибка при обновлении;
- случайное удаление конфигурации.

Поэтому после завершения настройки рекомендуется создать резервную копию всех важных данных.

---

# Что необходимо сохранить

Для полного восстановления системы достаточно сохранить несколько компонентов.

В данном проекте рекомендуется создать резервные копии:

- конфигурации сервера AmneziaWG;
- конфигураций всех клиентов;
- конфигурации OpenWrt;
- списков Podkop;
- пользовательских файлов маршрутизатора.

Если все эти данные сохранены, восстановление займёт значительно меньше времени, чем повторная настройка системы с нуля.

---

# Резервная копия VPS

Самыми важными являются конфигурационные файлы сервера.

Именно они содержат:

- приватные ключи;
- настройки интерфейса;
- список клиентов;
- параметры VPN.

Перед любыми серьёзными изменениями рекомендуется сохранить содержимое каталога конфигурации AmneziaWG.

Если используется панель управления, также рекомендуется сохранить её настройки.

---

# Экспорт конфигураций клиентов

Каждое устройство имеет собственную конфигурацию.

Например:

- Windows;
- iPhone;
- маршрутизатор Cudy.

Рекомендуется хранить эти конфигурации отдельно.

При утрате устройства можно быстро создать новую конфигурацию, не затрагивая остальных клиентов.

---

# Резервная копия OpenWrt

OpenWrt позволяет создать архив всей конфигурации.

В веб-интерфейсе LuCI откройте:

```
System
→ Backup / Flash Firmware
```

Выберите:

```
Generate Archive
```

Будет создан архив с конфигурацией маршрутизатора.

Его рекомендуется хранить в безопасном месте.

---

# Что входит в архив OpenWrt

Архив содержит:

- настройки сети;
- настройки LuCI;
- пользовательские конфигурации;
- файлы `/etc`;
- большинство установленных параметров.

После восстановления роутер сможет вернуться к прежнему состоянию значительно быстрее.

---

# Что архив не сохраняет

Следует понимать, что архив OpenWrt не является полной копией системы.

После восстановления может потребоваться повторная установка некоторых пакетов.

Поэтому полезно иметь отдельный список используемого программного обеспечения.

---

# Резервное копирование Podkop

Если используется Podkop, рекомендуется сохранить его конфигурацию отдельно.

В зависимости от способа установки это могут быть:

- пользовательские списки маршрутизации;
- файлы конфигурации;
- собственные правила.

При использовании Git значительно удобнее хранить такие файлы в отдельном репозитории.

Это позволит быстро восстановить изменения после переустановки OpenWrt.

---

# Резервное копирование sing-box

Если маршрутизация построена на основе sing-box, необходимо сохранить его конфигурационный файл.

Как правило, достаточно сохранить:

```
/etc/sing-box/config.json
```

Именно этот файл определяет работу маршрутизации и взаимодействие с AmneziaWG.

После восстановления системы достаточно вернуть этот файл на место.

---

# Сохранение списка установленных пакетов

После переустановки OpenWrt может потребоваться повторная установка пакетов.

Чтобы не вспоминать их вручную, рекомендуется заранее сохранить список:

```bash
opkg list-installed > installed-packages.txt
```

Полученный файл значительно ускорит восстановление системы.

---

# Что делать при переустановке VPS

Если сервер был полностью переустановлен, рекомендуется придерживаться следующего порядка.

1. Подготовить VPS.
2. Установить AmneziaWG.
3. Восстановить конфигурацию сервера.
4. Проверить работу интерфейса.
5. Проверить Handshake клиентов.
6. Убедиться в работе NAT.
7. Проверить доступ к панели управления.

Такой порядок позволяет быстрее обнаружить возможные ошибки.

---

# Что делать при переустановке OpenWrt

Если был сброшен или заменён маршрутизатор, последовательность действий будет следующей.

1. Установить OpenWrt.
2. Восстановить архив конфигурации.
3. Установить недостающие пакеты.
4. Вернуть конфигурацию sing-box.
5. Вернуть настройки Podkop.
6. Проверить интерфейс `awg0`.
7. Проверить Handshake.
8. Проверить работу маршрутизации.

После выполнения этих шагов роутер должен полностью восстановить свою работоспособность.

---

# Рекомендуемый чек-лист

Перед любыми серьёзными изменениями рекомендуется убедиться, что сохранены:

- ✅ конфигурация сервера AmneziaWG;
- ✅ конфигурации всех клиентов;
- ✅ архив OpenWrt;
- ✅ конфигурация sing-box;
- ✅ настройки Podkop;
- ✅ список установленных пакетов;
- ✅ резервные копии приватных ключей.

Если все пункты выполнены, восстановление системы займёт минимальное время.

---

# Полезные рекомендации

- Храните резервные копии в нескольких местах.
- Не размещайте приватные ключи в публичных репозиториях.
- После изменения конфигурации обновляйте резервные копии.
- Регулярно проверяйте возможность восстановления.
- Документируйте внесённые изменения.

Помните: резервная копия считается надёжной только тогда, когда её уже удалось успешно восстановить хотя бы один раз.
