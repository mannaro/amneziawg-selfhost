# 11. Диагностика и устранение неисправностей

## Введение

Даже при правильной настройке VPN иногда могут возникать проблемы.

Причинами могут быть:

- ошибки в конфигурации;
- проблемы с сетью;
- изменения на стороне провайдера;
- неправильные маршруты;
- отсутствие Handshake;
- ошибки после обновления OpenWrt или VPS.

В этой главе собраны наиболее распространённые неисправности и способы их диагностики.

---

# Общий алгоритм поиска неисправностей

Перед внесением изменений рекомендуется выполнять проверку в следующем порядке:

1. Проверить доступность VPS.
2. Проверить работу интерфейса AmneziaWG.
3. Проверить наличие Handshake.
4. Проверить маршрутизацию.
5. Проверить работу Podkop.
6. Проверить работу sing-box.
7. Проверить правила nftables.

Такой порядок позволяет значительно сократить время поиска проблемы.

---

# Проблема: отсутствует Handshake

Самый распространённый симптом — отсутствие Handshake.

Проверьте сервер:

```bash
sudo awg show
```

Если отображается:

```text
latest handshake: never
```

или время последнего Handshake не обновляется, соединение между клиентом и сервером не устанавливается.

### Возможные причины

- неверный Endpoint;
- закрыт UDP-порт;
- неправильный публичный IP;
- ошибка в ключах;
- блокировка UDP-трафика;
- неверное системное время.

---

# Проверка прослушиваемого порта

На VPS выполните:

```bash
sudo ss -lunp
```

Убедитесь, что AmneziaWG прослушивает необходимый UDP-порт.

Например:

```text
*:53925
```

Если порт отсутствует, проверьте настройки сервера и конфигурацию интерфейса.

---

# Проверка интерфейса

На сервере:

```bash
ip -br addr show awg0
```

Ожидаемый результат:

```text
awg0    UP    10.66.66.1/24
```

На роутере OpenWrt:

```bash
ip -br addr show awg0
```

Ожидаемый результат:

```text
awg0    UP    10.66.66.4/32
```

Если интерфейс отсутствует или находится в состоянии DOWN, VPN работать не сможет.

---

# Проблема: VPN подключён, но Интернет отсутствует

Если Handshake присутствует, но сайты не открываются, необходимо проверить маршрутизацию.

На VPS выполните:

```bash
sysctl net.ipv4.ip_forward
```

Ожидаемый результат:

```text
net.ipv4.ip_forward = 1
```

Затем убедитесь, что работает NAT.

В нашей конфигурации он реализован средствами nftables.

---

# Проверка маршрутизации OpenWrt

Проверьте правила Policy Routing:

```bash
ip rule
```

В выводе должно присутствовать правило:

```text
fwmark 0x100000 lookup podkop
```

Если правило отсутствует, трафик не сможет попасть в систему Podkop.

---

# Проверка таблицы маршрутизации

Если используется Podkop, необходимо убедиться, что существует отдельная таблица маршрутизации.

Проверьте её содержимое:

```bash
ip route show table podkop
```

Ожидаемый результат:

```text
local default dev lo scope host
```

Если таблица отсутствует или пуста, трафик не сможет попасть в TPROXY.

---

# Проверка TPROXY

TPROXY используется для передачи трафика локальному экземпляру sing-box.

Проверить наличие соответствующих правил можно командой:

```bash
nft list ruleset
```

Убедитесь, что в правилах присутствуют цепочки, связанные с Podkop и TPROXY.

Если соответствующие правила отсутствуют, пакеты не будут перенаправляться в sing-box.

---

# Проверка sing-box

Убедитесь, что процесс запущен.

```bash
ps | grep sing-box
```

Ожидаемый результат:

```text
/usr/bin/sing-box run -c /etc/sing-box/config.json
```

Также рекомендуется проверить прослушиваемый порт.

```bash
netstat -lntup | grep 1602
```

Ожидаемый результат:

```text
127.0.0.1:1602
```

Если процесс отсутствует или порт не прослушивается, маршрутизация через VPN работать не будет.

---

# Проверка nftables

Убедитесь, что правила загружены.

```bash
nft list ruleset
```

Если после изменения конфигурации правила отсутствуют, перезапустите соответствующий сервис или загрузите конфигурацию повторно.

---

# Проблема: VPN работает, но нужные сайты не используют туннель

Если VPN подключён, а отдельные сайты продолжают открываться напрямую, возможны следующие причины:

- домен отсутствует в списках Podkop;
- списки ещё не обновились;
- используется кэш DNS;
- соединение было установлено до применения новых правил.

Попробуйте:

- обновить списки;
- очистить DNS-кэш;
- переподключить VPN;
- повторно открыть сайт.

---

# Проблема: после перезагрузки роутера VPN не подключается

Проверьте:

- включён ли автоматический запуск интерфейса;
- запускается ли sing-box;
- присутствует ли интерфейс `awg0`;
- появляется ли Handshake.

Последовательно выполните:

```bash
ip -br addr show awg0
```

```bash
awg show
```

```bash
ps | grep sing-box
```

Эти три команды позволяют быстро определить, на каком этапе возникает проблема.

---

# Проблема: после переустановки VPS перестали подключаться клиенты

Если сервер был установлен заново, убедитесь, что восстановлены:

- приватные ключи;
- конфигурация AmneziaWG;
- список клиентов;
- правила nftables;
- NAT (Masquerade);
- IP Forwarding.

Если вместо восстановления была создана новая конфигурация сервера, старые клиентские файлы больше не смогут подключиться.

В этом случае потребуется создать новые конфигурации и импортировать их на все устройства.

---

# Чек-лист диагностики

Перед поиском сложных причин рекомендуется выполнить следующие проверки.

На VPS:

```bash
sudo awg show
```

```bash
ip -br addr show awg0
```

```bash
sysctl net.ipv4.ip_forward
```

```bash
nft list ruleset
```

На OpenWrt:

```bash
ip -br addr show awg0
```

```bash
awg show
```

```bash
ip rule
```

```bash
ip route show table podkop
```

```bash
ps | grep sing-box
```

```bash
netstat -lntup | grep 1602
```

В большинстве случаев именно эти команды позволяют быстро определить источник проблемы.

---

# Полезные рекомендации

- После изменения конфигурации всегда проверяйте Handshake.
- Не меняйте сразу несколько параметров одновременно.
- Перед обновлением OpenWrt создавайте резервную копию.
- Используйте отдельные конфигурации для каждого клиента.
- Не публикуйте приватные ключи и конфигурационные файлы.

При возникновении сложных проблем сначала сравните текущее состояние системы с рабочей конфигурацией, а затем вносите изменения по одному, проверяя результат после каждого шага.
