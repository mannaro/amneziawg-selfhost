# 2. Установка AmneziaWG

В этом разделе устанавливается сервер AmneziaWG при помощи проекта
[`wiresock/amneziawg-install`](https://github.com/wiresock/amneziawg-install).

Веб-панель устанавливается отдельно в следующей главе.

## Что будет создано

После завершения установки сервер будет содержать:

- интерфейс AmneziaWG `awg0`;
- VPN-подсеть `10.66.66.0/24`;
- адрес сервера `10.66.66.1`;
- UDP-порт для входящих VPN-подключений;
- systemd-службу `awg-quick@awg0`;
- основной конфигурационный файл;
- файл параметров установщика;
- первый клиентский профиль.

Основные серверные файлы размещаются в каталоге:

```text
/etc/amnezia/amneziawg
```

Клиентские конфигурации создаются в формате:

```text
/root/awg0-client-CLIENT_NAME.conf
```

> [!IMPORTANT]
> Клиентские `.conf` содержат приватные ключи. Их нельзя публиковать в GitHub,
> отправлять в общие чаты или хранить в открытых папках.

## Подключение к VPS

Подключитесь к чистому VPS:

```bash
ssh root@YOUR_SERVER_IP
```

Убедитесь, что вы действительно на сервере:

```bash
whoami
hostname
pwd
```

Ожидается пользователь:

```text
root
```

## Создание рабочего каталога

Чтобы не размещать установочные файлы непосредственно в `/root`, создайте
отдельный каталог:

```bash
mkdir -p /root/setup/installers
cd /root/setup/installers
```

## Клонирование репозитория

```bash
git clone https://github.com/wiresock/amneziawg-install.git
cd amneziawg-install
```

Проверьте содержимое:

```bash
ls -la
```

В каталоге должны присутствовать как минимум:

```text
amneziawg-install.sh
amneziawg-web.sh
README.md
```

## Фиксация используемой версии

Внешний репозиторий может меняться. Для воспроизводимой установки желательно
зафиксировать текущий commit или release/tag.

Посмотреть текущий commit:

```bash
git rev-parse HEAD
```

Посмотреть доступные теги:

```bash
git tag --sort=-version:refname | head
```

Если документация рассчитана на конкретную проверенную версию, переключитесь
на неё:

```bash
git checkout VERIFIED_TAG
```

Проверьте состояние:

```bash
git status
git describe --tags --always
```

> [!NOTE]
> В рабочей конфигурации использовалась версия веб-компонента
> `web-v0.1.8`. Перед повторной установкой нужно проверить, существует ли этот
> тег в текущем репозитории и относится ли он ко всему проекту или только к
> веб-панели.

## Проверка установочного скрипта

Сделайте файл исполняемым:

```bash
chmod +x amneziawg-install.sh
```

Проверьте первые строки скрипта:

```bash
head -n 20 amneziawg-install.sh
```

Дополнительно можно сохранить контрольную сумму:

```bash
sha256sum amneziawg-install.sh
```

Это позволит позже понять, использовалась ли та же версия файла.

## Запуск установки

```bash
./amneziawg-install.sh
```

Если вы вошли не как `root`, используйте:

```bash
sudo ./amneziawg-install.sh
```

Установщик проверит операционную систему, установит необходимые пакеты,
настроит AmneziaWG и предложит параметры сервера и первого клиента.

## Рекомендуемые параметры

Для конфигурации из этого руководства используются следующие значения:

| Параметр | Значение |
|---|---|
| VPN-интерфейс | `awg0` |
| IPv4 сервера внутри VPN | `10.66.66.1` |
| VPN-подсеть | `10.66.66.0/24` |
| Первый клиент | `10.66.66.2` |
| IPv6 | отключён или не используется |
| UDP-порт | любой свободный высокий порт |
| DNS клиента | выбранный публичный или собственный DNS |

В публичных примерах реальный порт обозначается как:

```text
AWG_PORT
```

## Split tunneling

Стандартный установщик может предложить:

```text
0.0.0.0/0, ::/0
```

Это означает полную маршрутизацию всего трафика клиента через VPN.

В нашей конфигурации используется split tunneling, поэтому итоговый
`AllowedIPs` позже заменяется списком нужных сетей и обязательно содержит
внутреннюю VPN-подсеть:

```text
10.66.66.0/24
```

> [!WARNING]
> Не удаляйте `10.66.66.0/24` из `AllowedIPs`. Без этого клиент может потерять
> доступ к VPN-адресу сервера и веб-панели.

Подробная генерация маршрутов описана в:

```text
docs/05-split-routing.md
```

На первом этапе допустимо завершить установку со стандартным значением, а
затем заменить его после проверки базового подключения.

## Что делает установщик

Установочный скрипт:

1. проверяет поддерживаемую операционную систему;
2. устанавливает AmneziaWG и необходимые системные компоненты;
3. генерирует серверный и клиентский ключи;
4. создаёт параметры обфускации;
5. формирует серверный конфигурационный файл;
6. включает IPv4 forwarding;
7. создаёт правила NAT;
8. создаёт и запускает systemd-службу;
9. формирует первый клиентский конфиг.

Параметры обфускации генерируются автоматически. Вручную подбирать `Jc`,
`Jmin`, `Jmax`, `S1`–`S4` и `H1`–`H4` обычно не требуется.

## Проверка файлов

После установки:

```bash
ls -la /etc/amnezia/amneziawg
```

Основные файлы:

```text
/etc/amnezia/amneziawg/awg0.conf
/etc/amnezia/amneziawg/params
```

Проверьте права:

```bash
stat -c '%a %U:%G %n' \
  /etc/amnezia/amneziawg/awg0.conf \
  /etc/amnezia/amneziawg/params
```

Файлы не должны быть доступны всем пользователям.

Не выводите содержимое `PrivateKey` в публичные журналы или скриншоты.

## Проверка systemd-службы

```bash
systemctl status awg-quick@awg0 --no-pager
```

Ожидается:

```text
Active: active (exited)
```

Для `awg-quick` состояние `active (exited)` является нормальным: служба
создаёт интерфейс и завершает установочный процесс, а сам интерфейс продолжает
работать в ядре.

Проверьте автозапуск:

```bash
systemctl is-enabled awg-quick@awg0
```

Ожидается:

```text
enabled
```

## Проверка интерфейса

```bash
ip -brief address show awg0
```

Ожидаемый адрес:

```text
10.66.66.1/24
```

Дополнительная проверка:

```bash
awg show
```

Должны отображаться:

- интерфейс `awg0`;
- публичный ключ сервера;
- UDP listen port;
- первый peer;
- `AllowedIPs` первого клиента.

## Проверка UDP-порта

Получить порт из активной конфигурации:

```bash
awg show awg0 listen-port
```

Проверить прослушивание UDP:

```bash
ss -lunp
```

Следует учитывать, что UDP-сокет не всегда отображается с понятным именем
процесса, поскольку AmneziaWG работает как модуль ядра.

## Проверка маршрутизации и NAT

IPv4 forwarding:

```bash
sysctl net.ipv4.ip_forward
```

Ожидается:

```text
net.ipv4.ip_forward = 1
```

Проверьте правила:

```bash
nft list ruleset
```

или, если установщик использует совместимые правила iptables:

```bash
iptables -t nat -S
iptables -S FORWARD
```

Должно присутствовать разрешение forwarding для VPN-подсети и masquerade
через внешний интерфейс VPS.

## Проверка первого клиентского файла

Найдите созданный профиль:

```bash
find /root -maxdepth 1 -type f -name 'awg0-client-*.conf' -ls
```

Проверьте права:

```bash
stat -c '%a %U:%G %n' /root/awg0-client-*.conf
```

Рекомендуемые права:

```text
600 root:root
```

При необходимости:

```bash
chmod 600 /root/awg0-client-*.conf
```

## Безопасное копирование клиента

Копирование выполняется с локального компьютера, а не из SSH-сеанса VPS.

Пример для WSL:

```bash
scp "root@YOUR_SERVER_IP:/root/awg0-client-CLIENT_NAME.conf" .
```

После импорта клиентского профиля храните файл в защищённом месте.

Не добавляйте его в репозиторий.

## Создание дополнительных клиентов

Повторно запустите скрипт:

```bash
cd /root/setup/installers/amneziawg-install
./amneziawg-install.sh
```

Установщик обнаружит существующую конфигурацию и откроет меню управления.

В актуальных версиях также могут использоваться неинтерактивные команды:

```bash
./amneziawg-install.sh --list-clients
./amneziawg-install.sh --add-client CLIENT_NAME
./amneziawg-install.sh --remove-client CLIENT_NAME
```

После создания каждого клиента проверяйте:

```bash
awg show
```

Адреса не должны повторяться.

Пример адресного плана:

| Устройство | VPN-адрес |
|---|---|
| Сервер | `10.66.66.1` |
| Windows PC | `10.66.66.2` |
| iPhone | `10.66.66.3` |
| Cudy/OpenWrt | `10.66.66.4` |

## Базовая проверка клиента

После импорта и включения профиля:

```bash
awg show
```

Для подключённого peer должны появиться:

```text
latest handshake: ...
transfer: ... received, ... sent
```

Если handshake отсутствует, проверьте:

1. правильность публичного IP VPS;
2. правильность UDP-порта;
3. доступность порта в firewall провайдера;
4. наличие маршрута до IP VPS;
5. совпадение параметров обфускации;
6. системное время клиента и сервера.

## Перезапуск интерфейса

```bash
systemctl restart awg-quick@awg0
```

После перезапуска:

```bash
systemctl status awg-quick@awg0 --no-pager
awg show
ip -brief address show awg0
```

## Журналы

```bash
journalctl -u awg-quick@awg0 --no-pager
```

Последние записи:

```bash
journalctl -u awg-quick@awg0 -n 100 --no-pager
```

Потоковое наблюдение:

```bash
journalctl -u awg-quick@awg0 -f
```

Для выхода нажмите:

```text
Ctrl+C
```

## Итоговая проверка

```bash
systemctl is-enabled awg-quick@awg0
systemctl is-active awg-quick@awg0
ip -brief address show awg0
awg show
sysctl net.ipv4.ip_forward
```

Перед следующим этапом должны выполняться условия:

- интерфейс `awg0` существует;
- сервер имеет адрес `10.66.66.1/24`;
- служба включена и активна;
- UDP-порт назначен;
- первый клиент присутствует в `awg show`;
- клиентский конфиг создан и защищён;
- NAT и IPv4 forwarding настроены;
- после подключения клиента появляется handshake.
