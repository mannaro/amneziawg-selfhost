# 02. Установка AmneziaWG

В этой главе устанавливается сервер **AmneziaWG** с использованием проекта **wiresock/amneziawg-install**.

После завершения установки сервер будет полностью готов к приёму VPN-подключений.

> **Важно**
>
> Веб-панель управления устанавливается отдельно и рассматривается в следующей главе.

---

# 🎯 Цель

В результате выполнения этой главы будет:

- установлен сервер AmneziaWG;
- создан VPN-интерфейс `awg0`;
- настроена VPN-подсеть `10.66.66.0/24`;
- создан первый клиент;
- подготовлена серверная конфигурация;
- настроен автоматический запуск службы.

---

# 📋 Что будет создано

После завершения установки сервер будет содержать:

- интерфейс `awg0`;
- VPN-подсеть `10.66.66.0/24`;
- адрес сервера `10.66.66.1`;
- UDP-порт для входящих VPN-подключений;
- службу `awg-quick@awg0`;
- основной конфигурационный файл;
- файл параметров установщика;
- первый клиентский профиль.

Основные серверные файлы размещаются в каталоге:

```text
/etc/amnezia/amneziawg
```

Клиентские конфигурации создаются в формате:

```text
/root/awg0-client-CLIENT_NAME.conf
```

---

# 🔒 Безопасность

Клиентские файлы `.conf` содержат приватные ключи.

Никогда не:

- публикуйте их в GitHub;
- отправляйте через открытые чаты;
- храните в общедоступных каталогах;
- прикладывайте к Issue или Pull Request.

Подключитесь к серверу:

```bash

```

Убедитесь, что вы действительно работаете на нужном сервере:

```bash
whoami
hostname
pwd
```

Ожидаемый пользователь:

```text
root
```

Чтобы не размещать установочные файлы непосредственно в `/root`, создайте отдельный каталог:

```bash

```

```

```

---

# ▶ Запуск установки

Запустите установщик:

```bash
./amneziawg-install.sh
```

Если используется обычный пользователь:

```bash
sudo ./amneziawg-install.sh
```

Во время установки автоматически выполняются следующие действия:

1. проверка поддерживаемой версии Ubuntu;
2. установка необходимых пакетов;
3. генерация серверных и клиентских ключей;
4. создание параметров обфускации;
5. генерация серверной конфигурации;
6. включение IPv4 Forwarding;
7. настройка NAT;
8. регистрация службы systemd;
9. создание первого клиентского профиля.

Параметры обфускации (`Jc`, `Jmin`, `Jmax`, `S1–S4`, `H1–H4`) генерируются автоматически. В большинстве случаев изменять их вручную не требуется.

---

# ⚙ Рекомендуемые параметры

Для конфигурации, используемой в этом руководстве, рекомендуется выбрать следующие значения:

| Параметр | Значение |
|----------|----------|
| VPN-интерфейс | `awg0` |
| Адрес сервера | `10.66.66.1` |
| VPN-подсеть | `10.66.66.0/24` |
| Первый клиент | `10.66.66.2` |
| IPv6 | отключён или не используется |
| UDP-порт | любой свободный высокий порт |
| DNS | публичный или собственный |

Во всех примерах документации реальный UDP-порт обозначается как:

```text
AWG_PORT
```
# 🌐 Split Tunneling

Во время установки скрипт может предложить значение:

```text
0.0.0.0/0, ::/0
```

Это означает, что **весь сетевой трафик клиента** будет направляться через VPN.

В данной документации используется **Split Tunneling**. После завершения базовой установки список `AllowedIPs` будет заменён только необходимыми маршрутами.

Минимально он должен содержать внутреннюю VPN-подсеть:

```text
10.66.66.0/24
```

> **Важно**
>
> Не удаляйте сеть `10.66.66.0/24` из `AllowedIPs`.
>
> В противном случае клиент потеряет доступ к VPN-адресу сервера, веб-панели и другим устройствам внутри VPN.

Подробная настройка Split Tunneling рассматривается в отдельной главе:

**[05. Split Routing](05-split-routing.md)**

На этапе первоначальной установки можно оставить значение, предложенное установщиком, а после проверки подключения заменить его на необходимый список маршрутов.

---

# 📂 Проверка созданных файлов

После завершения установки убедитесь, что основные файлы были созданы:

```bash
ls -la /etc/amnezia/amneziawg
```

Основные файлы:

```text
/etc/amnezia/amneziawg/awg0.conf
/etc/amnezia/amneziawg/params
```

Проверьте права доступа:

```bash
stat -c '%a %U:%G %n' \
  /etc/amnezia/amneziawg/awg0.conf \
  /etc/amnezia/amneziawg/params
```

Конфигурационные файлы не должны быть доступны другим пользователям системы.

> **Важно**
>
> Никогда не публикуйте содержимое поля `PrivateKey` в документации, логах, скриншотах или сообщениях.

---

# ⚙ Проверка службы systemd

Проверьте состояние службы:

```bash
systemctl status awg-quick@awg0 --no-pager
```

Ожидаемый результат:

```text
Active: active (exited)
```

Для `awg-quick` состояние **active (exited)** является нормальным.

Служба создаёт VPN-интерфейс, применяет конфигурацию и завершает выполнение, после чего интерфейс продолжает работать в ядре Linux.

Проверьте автоматический запуск:

```bash
systemctl is-enabled awg-quick@awg0
```

Ожидаемый результат:

```text
enabled
```

---

# 🌐 Проверка интерфейса

Убедитесь, что интерфейс создан:

```bash
ip -brief address show awg0
```

Ожидаемый адрес:

```text
10.66.66.1/24
```

Получите дополнительную информацию:

```bash
awg show
```

В выводе должны присутствовать:

- интерфейс `awg0`;
- публичный ключ сервера;
- UDP Listen Port;
- первый Peer;
- `AllowedIPs` первого клиента.

---

# 🔌 Проверка UDP-порта

Получить активный порт:

```bash
awg show awg0 listen-port
```

Проверить прослушиваемые UDP-порты:

```bash
ss -lunp
```

> **Примечание**
>
> UDP-сокет может отображаться без очевидного имени процесса, поскольку AmneziaWG работает как модуль ядра.

---

# 🔀 Проверка NAT и маршрутизации

Убедитесь, что IPv4 Forwarding включён:

```bash
sysctl net.ipv4.ip_forward
```

Ожидаемый результат:

```text
net.ipv4.ip_forward = 1
```

Проверьте правила Firewall:

```bash
nft list ruleset
```

Если используются совместимые правила `iptables`:

```bash
iptables -t nat -S
iptables -S FORWARD
```

В правилах должны присутствовать:

- разрешение пересылки пакетов (`FORWARD`);
- правило `masquerade` для внешнего интерфейса;
- обработка VPN-подсети.

---

# 👤 Проверка клиентского профиля

Найдите созданный конфигурационный файл:

```bash
find /root -maxdepth 1 -type f -name 'awg0-client-*.conf' -ls
```

Проверьте права доступа:

```bash
stat -c '%a %U:%G %n' /root/awg0-client-*.conf
```

Рекомендуемые права:

```text
600 root:root
```

При необходимости исправьте их:

```bash
chmod 600 /root/awg0-client-*.conf
```

---

# 📤 Безопасное копирование клиента

Копирование выполняется **с локального компьютера**, а не из SSH-сеанса VPS.

Пример для Linux или WSL:

```bash
scp "root@YOUR_SERVER_IP:/root/awg0-client-CLIENT_NAME.conf" .
```

После импорта профиля:

- сохраните файл в защищённом месте;
- создайте резервную копию;
- не публикуйте его в репозитории.

---

# 👥 Создание дополнительных клиентов

Для добавления новых клиентов повторно запустите установочный скрипт:

```bash
cd /root/setup/installers/amneziawg-install
./amneziawg-install.sh
```

Если сервер уже настроен, установщик обнаружит существующую конфигурацию и откроет меню управления.

В некоторых версиях также доступны неинтерактивные команды:

```bash
./amneziawg-install.sh --list-clients
./amneziawg-install.sh --add-client CLIENT_NAME
./amneziawg-install.sh --remove-client CLIENT_NAME
```

> **Примечание**
>
> Набор доступных команд зависит от версии проекта. Если какая-либо команда отсутствует, используйте интерактивное меню установщика.

После создания каждого клиента рекомендуется выполнить:

```bash
awg show
```

Убедитесь, что каждому устройству назначен уникальный VPN-адрес.

Пример адресного плана:

| Устройство | VPN-адрес |
|------------|-----------|
| Сервер | `10.66.66.1` |
| Windows PC | `10.66.66.2` |
| iPhone | `10.66.66.3` |
| Cudy / OpenWrt | `10.66.66.4` |

---

# 🔗 Проверка подключения клиента

После импорта клиентского профиля подключитесь к VPN и выполните на сервере:

```bash
awg show
```

Для подключённого клиента должны появиться строки примерно следующего вида:

```text
latest handshake: ...
transfer: ... received, ... sent
```

Наличие `latest handshake` означает успешный обмен ключами между клиентом и сервером.

Если соединение не устанавливается, проверьте:

- корректность публичного IP-адреса VPS;
- правильность UDP-порта;
- открытие UDP-порта в firewall VPS и панели провайдера;
- доступность маршрута до сервера;
- совпадение параметров обфускации;
- корректность системного времени на сервере и клиенте.

```bash
systemctl status awg-quick@awg0 --no-pager
```

awg show

# 📜 Просмотр журналов

При возникновении проблем используйте журнал службы.

Полный журнал:

```bash
journalctl -u awg-quick@awg0 --no-pager
```

Последние 100 записей:

```bash
journalctl -u awg-quick@awg0 -n 100 --no-pager
```

Просмотр журнала в реальном времени:

```bash
journalctl -u awg-quick@awg0 -f
```

Для выхода нажмите:

```text
Ctrl+C
```
