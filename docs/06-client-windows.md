# 06. Клиент AmneziaWG для Windows

## Введение

После подготовки VPS и настройки сервера необходимо подключить первый клиент.

В этой главе будет рассмотрена установка клиента AmneziaWG на Windows, импорт конфигурации и проверка правильности работы VPN.

В результате выполнения всех шагов вы сможете:

- подключаться к своему VPN одним нажатием;
- проверить успешное соединение с сервером;
- убедиться в работе Split Routing;
- выполнить базовую диагностику при возникновении проблем.

---

# Скачивание клиента

Актуальную версию клиента можно скачать с официального сайта проекта Amnezia.

Рекомендуется использовать последнюю стабильную версию для Windows.

После установки запустите программу.

При первом запуске окно будет практически пустым — это нормально.

---

# Получение конфигурации

Каждое устройство использует собственную конфигурацию.

Не рекомендуется использовать один и тот же конфигурационный файл сразу на нескольких устройствах.

Это значительно усложняет диагностику и снижает безопасность.

Конфигурацию можно получить двумя способами:

- через веб-панель AmneziaWG;
- через QR-код (для мобильных устройств).

Для Windows используется обычный файл конфигурации.

---

# Импорт конфигурации

Нажмите кнопку импорта конфигурации.

Выберите файл, полученный с сервера.

После успешного импорта в списке появится новое VPN-подключение.

До подключения его состояние будет отображаться как отключённое.

---

# Первое подключение

Нажмите кнопку подключения.

Через несколько секунд статус должен измениться на **Подключено**.

Если подключение не устанавливается, не стоит сразу искать проблему на стороне клиента.

Большинство ошибок можно быстро обнаружить несколькими проверками, которые будут рассмотрены ниже.

---

# Проверка соединения

Первое, что необходимо проверить, — действительно ли клиент подключился к серверу.

На VPS выполните команду:

```bash
sudo awg show
```

В выводе должна появиться информация о клиенте.

Например:

```text
peer: XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
endpoint: <публичный IP клиента>:порт
latest handshake: 5 seconds ago
transfer: 10.3 MiB received, 41.8 MiB sent
```

Особенно важны две строки:

- `latest handshake`;
- `transfer`.

Если время постоянно обновляется, соединение установлено успешно.

Если значение отсутствует или отображается как несколько часов назад, клиент до сервера не достучался.

---

# Проверка доступа в Интернет

После подключения убедитесь, что Интернет продолжает работать.

Откройте любой сайт.

Если страницы не открываются:

- проверьте соединение с VPS;
- убедитесь, что сервер выполняет NAT;
- проверьте настройки файрвола.

В данной конфигурации сервер автоматически выполняет masquerade с помощью nftables, поэтому дополнительная настройка клиента не требуется.

---

# Проверка внешнего IP

Самый простой способ убедиться, что VPN работает корректно, — проверить внешний IP-адрес.

До подключения запишите текущий IP.

После подключения снова проверьте его.

Если используется Full Tunnel, внешний IP изменится на адрес VPS.

Если используется Split Routing, внешний IP может остаться прежним — это зависит от того, какие маршруты используются клиентом.

Именно такой режим используется в данном проекте.

---

# Проверка Split Routing

После подключения Windows автоматически добавляет маршруты для сетей, которые должны использовать VPN.

Основной маршрут системы при этом остаётся без изменений.

Именно благодаря этому обычный интернет продолжает работать напрямую, а VPN используется только для необходимых сетей.

Проверить таблицу маршрутизации можно командой:

```powershell
route print
```

В выводе появятся дополнительные маршруты через интерфейс AmneziaWG.

Основной шлюз при этом останется вашим домашним маршрутизатором.

Это означает, что Split Routing работает корректно.

---

# Проверка доступа к панели управления

В данной конфигурации веб-панель AmneziaWG доступна только через VPN.

После подключения откройте браузер и перейдите по адресу:

```
http://10.66.66.1:8080
```

Если всё настроено правильно, откроется панель управления сервером.

Если страница недоступна:

- убедитесь, что VPN подключён;
- проверьте адрес клиента;
- убедитесь, что ваш IP разрешён в правилах nftables сервера.

---

# Проверка работы VPN

После первого успешного подключения рекомендуется выполнить несколько простых проверок.

## Проверка Handshake

На VPS выполните:

```bash
sudo awg show
```

В информации о вашем клиенте должны регулярно обновляться две строки:

```text
latest handshake: 12 seconds ago
transfer: 58.4 MiB received, 137 MiB sent
```

Если значение `latest handshake` постоянно обновляется, VPN работает корректно.

Если вместо этого отображается:

```text
latest handshake: never
```

или прошло несколько часов с момента последнего подключения, значит клиент не может установить соединение с сервером.

---

## Проверка доступа к серверу

После подключения попробуйте открыть веб-панель AmneziaWG.

В данной конфигурации она доступна только через VPN.

Адрес панели:

```
http://10.66.66.1:8080
```

Если страница открылась — соединение полностью работоспособно.

Если браузер сообщает, что соединение невозможно установить, проверьте:

- подключён ли VPN;
- разрешён ли ваш клиент в правилах nftables;
- работает ли интерфейс `awg0` на VPS.

---

## Проверка маршрутов Windows

Откройте командную строку Windows и выполните:

```cmd
route print
```

В таблице маршрутизации должны появиться дополнительные сети, использующие интерфейс AmneziaWG.

При этом маршрут по умолчанию останется прежним.

Это означает, что используется Split Routing.

---

## Проверка внешнего IP

Откройте любой сервис проверки IP-адреса.

Если используется Full Tunnel, внешний IP изменится на адрес VPS.

В данной конфигурации используется Split Routing, поэтому изменение IP зависит от того, какой ресурс вы открываете.

Это нормальное поведение.

---

# Типичные проблемы

## Нет Handshake

Причины могут быть следующими:

- VPS недоступен;
- закрыт UDP-порт;
- неверный публичный IP сервера;
- неправильный ключ клиента;
- блокировка со стороны провайдера.

Проверьте:

```bash
sudo awg show
```

и

```bash
sudo ss -lunp | grep 53925
```

---

## Интернет перестал работать

Обычно причиной являются:

- неправильные маршруты;
- ошибка в конфигурации клиента;
- отсутствие NAT на сервере.

Проверьте работу интерфейса:

```bash
ip -br addr show awg0
```

И убедитесь, что на VPS включена маршрутизация:

```bash
sysctl net.ipv4.ip_forward
```

Ожидаемый результат:

```text
net.ipv4.ip_forward = 1
```

---

## Панель управления не открывается

Если VPN подключён, но страница `http://10.66.66.1:8080` недоступна, проверьте правила файрвола.

В данной конфигурации доступ к панели разрешён только определённым VPN-клиентам.

Если IP-адрес клиента отсутствует в правилах nftables, соединение будет отклонено.

---

## VPN подключается, но сайты открываются напрямую

Это не является ошибкой.

В проекте используется Split Routing.

Через VPN проходят только те сети, которые определены конфигурацией клиента или системой Podkop.

Остальной трафик продолжает использовать обычное интернет-подключение.

---

# Полезные рекомендации

- Не используйте один конфигурационный файл на нескольких устройствах.
- Для каждого клиента создавайте отдельную конфигурацию.
- Не публикуйте приватные ключи и Preshared Key.
- Регулярно проверяйте наличие Handshake после изменения конфигурации сервера.
- Перед внесением изменений в настройки VPN создавайте резервную копию конфигурации.

---

## Подготовка списка AllowedIPs

Для выборочной маршрутизации Windows-клиенту требуется список IP-адресов и подсетей.

Сформировать исходный список можно на странице:

[IP Address Collection and Management Service](https://iplist.opencck.org/ru/)

После загрузки файла рекомендуется проверить его содержимое:

```bash
head -n 20 networks.txt
```

Удалить пустые строки:

```bash
sed '/^[[:space:]]*$/d' networks.txt > networks-clean.txt
```

Удалить повторяющиеся записи:

```bash
sort -u networks-clean.txt > networks-unique.txt
```

Объединить подсети в строку для `AllowedIPs`:

```bash
paste -sd, networks-unique.txt | sed 's/,/, /g' > allowedips.txt
```

Проверить результат:

```bash
batcat allowedips.txt
```

Полученную строку вставьте в клиентскую конфигурацию:

```ini
[Peer]
PublicKey = SERVER_PUBLIC_KEY
PresharedKey = CLIENT_PRESHARED_KEY
Endpoint = SERVER_IP:SERVER_PORT
AllowedIPs = NETWORK_1, NETWORK_2, NETWORK_3
PersistentKeepalive = 25
```

После сохранения конфигурации переподключите туннель.

---

## Проверка выборочной маршрутизации

После подключения убедитесь, что маршрут для выбранного IP проходит через VPN:

```powershell
route print
```

Для проверки конкретного адреса можно использовать:

```powershell
tracert TARGET_IP
```

Также сравните внешний IP:

1. при обращении к ресурсу из списка;
2. при обращении к обычному ресурсу, который не включён в `AllowedIPs`.

Ожидаемое поведение:

- выбранный трафик проходит через VPS;
- остальной трафик использует обычное интернет-подключение;
- локальная сеть остаётся доступной.

!!! warning "Не используйте 0.0.0.0/0 для выборочного режима"

    Значение:

    ```ini
    AllowedIPs = 0.0.0.0/0
    ```

    отправляет через туннель весь IPv4-трафик.

    Для Split Routing указываются только необходимые адреса и подсети.
