# 07. Клиент AmneziaWG для iPhone (iOS)

## Введение

После настройки Windows-клиента можно подключить мобильные устройства.

В этой главе рассматривается настройка клиента AmneziaWG на iPhone с использованием QR-кода, создание VPN-подключения и проверка его работы.

После выполнения всех шагов вы сможете:

- подключаться к своему VPN с iPhone;
- быстро импортировать конфигурацию через QR-код;
- проверить успешное подключение;
- диагностировать наиболее распространённые проблемы.

---

# Установка приложения

Откройте App Store и установите приложение **AmneziaVPN**.

После установки запустите приложение.

При первом запуске список подключений будет пустым.

Это нормально.

---

# Создание конфигурации

Для каждого устройства рекомендуется создавать отдельного клиента.

Не используйте одну конфигурацию одновременно на нескольких устройствах.

Это усложняет диагностику и не соответствует хорошим практикам безопасности.

Откройте веб-панель AmneziaWG и создайте нового клиента.

После создания выберите отображение QR-кода.

---

# Импорт через QR-код

В приложении AmneziaVPN выберите:

```
Добавить подключение
```

Затем выберите:

```
Сканировать QR-код
```

Наведите камеру iPhone на QR-код, отображаемый в панели управления.

Через несколько секунд конфигурация будет импортирована автоматически.

---

# Создание VPN

После импорта iOS предложит добавить VPN-конфигурацию.

Нажмите:

```
Разрешить
```

После этого потребуется подтверждение с помощью Face ID, Touch ID или пароля устройства.

Это стандартный механизм безопасности iOS.

---

# Первое подключение

В приложении нажмите кнопку подключения.

Через несколько секунд статус должен измениться на **Подключено**.

Одновременно в строке состояния iPhone появится значок VPN.

Это означает, что система успешно подключилась к серверу.

---

# Проверка Handshake

На VPS выполните:

```bash
sudo awg show
```

В списке клиентов должен появиться новый Peer.

Пример:

```text
latest handshake: 8 seconds ago
transfer: 3.2 MiB received, 5.8 MiB sent
```

Если значение Handshake постоянно обновляется, соединение работает корректно.

---

# Проверка доступа к панели

Откройте Safari и перейдите по адресу:

```
http://10.66.66.1:8080
```

Если панель открылась, VPN функционирует правильно.

Если соединение отсутствует, сначала убедитесь, что VPN действительно подключён.

---

# Работа в фоновом режиме

iOS автоматически управляет VPN-подключениями.

При блокировке экрана VPN продолжает работать.

После выхода устройства из режима сна соединение может автоматически восстановиться без участия пользователя.

Это нормальное поведение системы.

---

# Типичные проблемы

## Не импортируется QR-код

Проверьте:

- яркость экрана компьютера;
- качество изображения;
- расстояние до камеры.

Если проблема сохраняется, можно импортировать конфигурацию вручную.

---

## Нет Handshake

Если сервер показывает:

```text
latest handshake: never
```

проверьте:

- доступность VPS;
- правильность конфигурации;
- наличие подключения к Интернету.

---

## Панель управления недоступна

Убедитесь, что:

- VPN подключён;
- клиент получил правильную конфигурацию;
- IP клиента разрешён правилами nftables на сервере.

---

## Интернет работает без VPN

В данной документации используется Split Routing.

Это означает, что только определённый трафик проходит через VPN.

Остальные соединения продолжают использовать обычный Интернет.

Такое поведение является ожидаемым.

---

# Полезные рекомендации

- Создавайте отдельную конфигурацию для каждого устройства.
- Не пересылайте QR-код другим пользователям.
- После удаления клиента из панели управления создайте новый QR-код.
- Не публикуйте конфигурационные файлы и приватные ключи.
